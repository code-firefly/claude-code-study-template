## 更新进度

**触发方式**（任意一种）：
- "更新进度 `<模块名>`" / "更新 `<模块名>` 的学习进度" / "同步进度 `<模块名>`"
- "记录进度 `<模块名>`" / "保存进度 `<模块名>`" / "打勾 `<模块名>`"
- "标记完成项 `<模块名>`" / "更新 `<模块名>` 状态" / "刷新进度 `<模块名>`"
- "`<模块名>` 进度更新" / "帮我更新 `<模块名>` 进度" / "`<模块名>` 学了多少"

**执行步骤**：

### 步骤 0：非阻塞更新检查
1. 从 CHANGELOG.md 提取本地版本（查找 `## [版本号]` 模式）
2. 尝试从 upstream 获取上游版本（如配置了 upstream）
3. 如上游版本更新，存储更新提醒字符串（格式如下）：
   ```
   ╔════════════════════════════════════════════════════════════════╗
   ║  📢 发现新版本                                                 ║
   ║                                                                ║
   ║  当前版本: v2.0.1  |  上游版本: v2.1.0                        ║
   ║                                                                ║
   ║  建议：学习结束后运行 "同步学习计划" 获取最新功能             ║
   ╚════════════════════════════════════════════════════════════════╝

   [以下是命令结果...]
   ```
4. 在最终输出前，如存在更新提醒，则添加到结果开头

### 步骤 1：读取模块的 `checklist.md`

### 步骤 2：解析学习模式和完成项：
   - 检测已勾选的学习模式（快速/完整）
   - 统计已完成的 checkbox 数量
   - 根据学习模式计算对应的完成百分比

### 步骤 3：读取 `PROGRESS.md` 获取当前进度

### 步骤 4：生成更新预览：
   ```
   Progress: X% -> Y%
   Status: 进行中
   进度条: ██████░░░░░░░░░░░░░░ 30%
   ```

### 步骤 5：用户确认后更新 `PROGRESS.md`：
   - 更新模块的完成百分比
   - 添加学习日志条目

**进度计算规则**：
- 快速模式：只统计"📚 快速模式"章节下的 checkbox
- 完整模式：统计"🛠️ 完整模式"章节下的所有 checkbox
- 未选择模式：基于完整模式计算
