## 自定义模块标记（Fork 用户保护机制）

### .custom 标记文件

**用途**：标识用户已自定义的模块，在同步更新时跳过或特殊处理

**位置**：`模块目录/.custom`

**格式**：
```markdown
# 模块自定义标记

此模块包含用户自定义内容，将在同步更新时被跳过或需要特殊处理。

- **自定义日期**：2026-03-07
- **自定义文件**：README.md, checklist.md, notes.md
- **自定义说明**：根据个人学习需求调整了课程内容

---
创建时间：2026-03-07T10:30:00Z
创建方式：learning-manager 自动创建
```

### 自动创建 .custom 标记

在以下情况自动创建 .custom 标记：

1. **用户编辑模块内容时**：
   - 编辑模块的 README.md
   - 修改模块的 checklist.md（超出学习进度更新的范围）
   - 更新模块的 notes.md（超出学习笔记记录的范围）

2. **创建自定义提示**：
   ```
   ⚠️  检测到您正在编辑模块内容
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   这可能意味着您在自定义课程内容以适应个人学习需求。

   已为此模块创建 .custom 标记，在同步更新时：
   - 此模块将被跳过，避免您的自定义内容被覆盖
   - 您可以选择备份自定义内容或手动合并上游更新

   模块：ai-tools-fundamentals
   标记文件：01-基础入门/ai-tools-fundamentals/.custom
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   ```

### 自定义状态警告

在操作已标记的模块时显示警告：

```
⚠️  自定义模块警告
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
此模块包含自定义内容（.custom 标记）

上次自定义：2026-03-07
自定义文件：README.md, checklist.md

建议：
- 同步更新时此模块将被跳过
- 定期备份自定义内容到 .backups/
- 需要上游更新时可手动选择合并策略
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 移除 .custom 标记

用户可以通过以下方式移除 .custom 标记：
- 删除模块目录下的 `.custom` 文件
- 或说 "移除 `<模块名>` 的自定义标记"

## 开始学习

**触发方式**（任意一种）：
- "开始学习 `<模块名>`" / "启动 `<模块名>` 的学习" / "开始学习模块 `<模块名>`"
- "我要学 `<模块名>`" / "我要开始学 `<模块名>`" / "我想学 `<模块名>`"
- "开始 `<模块名>`" / "进入 `<模块名>` 模块" / "学习 `<模块名>`"
- "学 `<模块名>`" / "启动 `<模块名>`" / "开启 `<模块名>` 学习"
- "`<模块名>` 开始学习" / "对 `<模块名>` 开始学习"

**执行步骤**：

### 步骤 0：非阻塞更新检查（带缓存机制）

**配置文件位置**：`.claude/update-config.json`

**执行逻辑**：
1. **加载配置管理工具**：
   - 如不存在，自动初始化默认配置（24小时检查间隔）
   - 检查 `jq` 工具是否可用（不可用时降级为无缓存模式）

2. **判断是否需要网络检查**：
   - 检查距离上次检查是否超过 `intervalHours`（默认24小时）
   - **在时间窗口内**：使用缓存的版本号，跳过 `git fetch upstream`
   - **超出时间窗口**：执行网络检查并更新缓存

3. **版本比较与提醒存储**：
   - 从 CHANGELOG.md 提取本地版本
   - 从缓存或 upstream 获取上游版本
   - 如上游版本更新，存储更新提醒字符串：
   ```
   ╔════════════════════════════════════════════════════════════════╗
   ║  📢 发现新版本                                                 ║
   ║                                                                ║
   ║  当前版本: v2.0.1  |  上游版本: v2.1.0                        ║
   ║  缓存时间: 2026-03-07T10:30:00Z（24小时内不重复检查）          ║
   ║                                                                ║
   ║  建议：学习结束后运行 "同步学习计划" 获取最新功能             ║
   ╚════════════════════════════════════════════════════════════════╝

   [以下是命令结果...]
   ```

4. **显示缓存状态**（可选）：
   - 如使用缓存版本，在提醒中注明"（缓存版本）"
   - 如首次运行或配置文件不存在，显示一次性提示告知新功能

5. 在最终输出前，如存在更新提醒，则添加到结果开头

### 步骤 0.5：检查知识缓存
1. 读取模块的 `knowledge/.metadata.json`（如果存在）
2. **情况 A：缓存存在**
   - 显示：`发现本地缓存 (缓存日期: YYYY-MM-DD)`
   - 询问用户：
     - 1. 使用本地缓存（快速开始）
     - 2. 更新缓存（拉取最新资料）
3. **情况 B：缓存不存在**
   - 显示：`未找到知识缓存`
   - 询问用户：
     - 1. 初始化缓存（推荐，首次拉取）
     - 2. 跳过缓存（直接开始学习）

### 步骤 1：模块名称别名解析
如果用户输入的模块名不在映射表中，检查别名映射：

| 旧名称 | 新名称 | 说明 |
|--------|--------|------|
| `claude-code-core` | `ai-tools-fundamentals` | AI 工具基础 |
| `mcp-basics` | `mcp-protocol` | MCP 协议入门 |
| `agent-sdk` | `agent-configuration` | Agent 配置与使用 |
| `mcp-advanced` | `mcp-advanced-config` | MCP 高级配置 |
| `openclaw-ecosystem` | `ai-orchestration` | AI 编排平台 |
| `everything-claude-code` | `ai-resources-research` | AI 资源研究 |
| `cc-switch` | `config-management` | 配置管理工具 |
| `spec-kit` | `spec-driven-dev` | 规范驱动开发 |
| `projects` | `practical-projects` | 综合实战项目 |

如果找到别名：
- 显示提示：`"注意：'{旧名称}' 已更名为 '{新名称}'，正在使用新名称继续..."`
- 使用新名称继续后续流程

### 步骤 2：查找模块路径
从映射表查找模块路径，优先匹配新名称。

### 步骤 3：验证模块状态
检查模块是否已开始学习（检查 checklist.md 的学习模式选择）
- 如果已开始，显示警告信息

### 步骤 4：选择学习模式
询问用户选择：
- 1. 📚 快速模式（约 3 天）- 理论为主
- 2. 🛠️ 完整模式（约 1 周）- 理论+实践

### 步骤 5：显示更新预览
- 模块信息（名称、优先级、模式、开始日期）
- 将要更新的文件列表
- 等待用户确认

### 步骤 6：更新文件
用户确认后，更新以下文件：
- **checklist.md**：勾选对应的学习模式 checkbox
- **notes.md**：设置学习模式和开始日期
- **PROGRESS.md**：更新模块状态为"进行中"，添加学习日志

### 步骤 7：显示成功信息
显示成功信息和下一步指导

## 重置模块

**触发方式**（任意一种）：
- "重置模块 `<模块名>`" / "重新开始 `<模块名>`" / "清除 `<模块名>` 的进度"
- "重学 `<模块名>`" / "`<模块名>` 重新开始" / "从头开始 `<模块名>`"
- "清除进度 `<模块名>`" / "重置 `<模块名>`" / "`<模块名>` 清零"

**执行步骤**：

1. 显示警告：此操作将清除模块的所有学习进度

2. 询问用户确认

3. 重置以下文件：
   - **checklist.md**：取消所有勾选项，清除学习模式选择
   - **notes.md**：重置日期和内容
   - **PROGRESS.md**：状态改为"未开始"，进度改为 0%

4. 添加重置日志

## 文件操作模式

### checklist.md 更新模式

```markdown
# 修改前
- [ ] **快速模式**（约 3 天）- 理论为主
- [x] **完整模式**（约 1 周）- 理论+实践

# 修改为（快速模式）
- [x] **快速模式**（约 3 天）- 理论为主
- [ ] **完整模式**（约 1 周）- 理论+实践
```

### notes.md 更新模式

```markdown
# 修改前
> **学习模式**：待选择
> **开始日期**：待定

# 修改为（开始快速模式学习）
> **学习模式**：快速模式
> **开始日期**：2026-03-07
```

### PROGRESS.md 更新模式

```markdown
# 修改模块行
# 从：
| ai-tools-fundamentals | P0 🔴 | 未选择 | 未开始 | 0% |

# 改为（开始快速模式）：
| ai-tools-fundamentals | P0 🔴 | 快速模式 | 进行中 | 0% |

# 添加学习日志
| 2026-03-07 | ai-tools-fundamentals | 开始学习 | 选择快速模式 |
```

## 注意事项

1. **用户确认**：修改多个文件前，总是先显示预览并等待用户确认
2. **日期格式**：统一使用 YYYY-MM-DD 格式
3. **文件路径**：所有路径相对于仓库根目录
4. **错误处理**：如果文件不存在或模块未找到，给出明确的错误提示
5. **进度计算**：基于实际勾选的 checkbox 数量，不要估算
