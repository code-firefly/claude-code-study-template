# Learning Sync Skill

## 功能范围
- 检查上游更新（版本比较）
- 同步学习计划（备份、拉取、合并）
- 新增/更新/删除模块

---

## 检查更新

**触发方式**（任意一种）：
- "检查更新" / "查看是否有更新" / "有更新吗"
- "模板有更新吗" / "检查新版本" / "版本更新"

**执行步骤**：

1. 检查 upstream 远程仓库是否配置
   - 未配置：显示配置指引并退出

2. 从 CHANGELOG.md 提取当前版本和上游版本

3. 显示版本比较结果：
   - 如有更新：显示版本号差异和同步指引
   - 如已是最新：显示相应提示

---

## 同步学习计划

**触发方式**（任意一种）：
- "同步学习计划" / "更新学习计划" / "同步更新"
- "检查模板更新" / "拉取更新" / "上游更新"

**执行步骤**：

1. 检查更新（同 check 模式）

2. 如有更新，显示同步预览：
   - 当前版本和上游版本
   - 同步流程说明（备份、获取、合并、验证）
   - 个人数据保护说明

3. 用户确认后，调用 `scripts/sync.sh` 执行同步：
   - 备份个人数据到 `.backups/sync-<日期>/`
   - 从 upstream 获取最新更新
   - 合并更新到本地分支
   - 验证个人数据完整性

4. 显示同步结果和后续指引

### 个人数据保护

以下文件已被 `.gitignore` 保护，不会被覆盖：
- `PROGRESS.md`
- `*/checklist.md`
- `*/notes.md`
- `.claude/LEARNING_BOOKMARKS.md`
- `.claude/KNOWLEDGE_CACHE.md`

## 新增模块

**触发方式**（任意一种）：
- "新增模块 `<模块名>`" / "添加模块 `<模块名>`" / "创建新模块 `<模块名>`"
- "新建模块 `<模块名>`" / "创建模块 `<模块名>`" / "加个模块 `<模块名>`"
- "添加一个模块" / "新建一个学习模块" / "创建模块"

**执行步骤**：

1. 询问用户确认模块信息：
   - 模块名称（英文短名称，如 `my-new-module`）
   - 所属阶段（01-基础入门/02-进阶探索/03-实战应用）
   - 优先级（P0 必学/P1 推荐/P2 可选/P3 了解）
   - 预计时长（完整模式）
   - 模式支持（快速/完整/两者）

2. 显示预览，等待用户确认

3. 创建模块目录和文件：
   - 创建 `XX-阶段名称/模块名/` 目录
   - 生成 `README.md`（模块导学）
   - 生成 `checklist.md`（学习清单模板）
   - 生成 `notes.md`（学习笔记模板）

4. 更新以下文件：
   - **PROGRESS.md**：添加模块行到对应阶段的表格
   - **CLAUDE.md**：添加模块路径映射到"开始学习"章节

5. 添加系统日志

---

## 更新模块信息

**触发方式**（任意一种）：
- "更新模块信息 `<模块名>`" / "修改模块 `<模块名>`" / "编辑模块 `<模块名>`"
- "更改模块 `<模块名>`" / "调整模块 `<模块名>`" / "`<模块名>` 信息修改"

**执行步骤**：

1. 读取模块当前信息

2. 询问用户要修改的字段：
   - 名称
   - 优先级
   - 预计时长
   - 模式支持

3. 显示变更预览

4. 更新相关文件：
   - `XX-阶段名称/模块名/README.md`
   - `PROGRESS.md`
   - `CLAUDE.md`（如名称或路径变更）

---

## 删除模块

**触发方式**（任意一种）：
- "删除模块 `<模块名>`" / "移除模块 `<模块名>`" / "去掉模块 `<模块名>`"
- "卸载模块 `<模块名>`" / "`<模块名>` 不要了" / "删掉 `<模块名>`"

**执行步骤**：

1. 显示警告：此操作将删除模块目录和所有相关文件

2. 询问用户确认（需要输入模块名确认）

3. 执行删除：
   - 删除模块目录及其所有文件
   - 从 PROGRESS.md 移除模块行
   - 从 CLAUDE.md 移除模块映射

4. 添加删除日志


## 注意事项

1. **用户确认**：删除模块前需要输入模块名确认
2. **备份策略**：同步前自动备份到 `.backups/sync-<日期>/`
3. **数据保护**：个人数据文件被 `.gitignore` 保护
4. **日期格式**：统一使用 YYYY-MM-DD 格式
5. **文件路径**：所有路径相对于仓库根目录

## 同步脚本

调用 `scripts/sync.sh` 执行实际同步操作：
- 备份个人数据
- 拉取上游更新
- 合并到本地
- 验证完整性
