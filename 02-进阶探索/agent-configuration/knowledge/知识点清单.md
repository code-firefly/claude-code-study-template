## 📋 知识点总览

### 一、核心概念 (4 个)

#### 1. Agent vs Chatbot 的本质区别
- **被动响应 vs 自主决策**：Chatbot 等待用户输入，Agent 主动规划执行
- **单轮交互 vs 多步骤推理**：Chatbot 一次性回答，Agent 分步执行复杂任务
- **能力边界差异**：Agent 具备工具调用能力和 API 访问权限
- **运行机制差异**：Chatbot 是请求-响应，Agent 是循环控制直到任务完成

**关键洞察**：从"问答系统"到"任务执行系统"的转变

#### 2. Agent 四大核心能力
| 能力 | 说明 | 实现方式 |
|------|------|----------|
| **工具调用** | Claude 根据需要调用预定义工具 | `betaZodTool` + `toolRunner` |
| **循环控制** | 持续评估任务状态，未完成则继续 | `toolRunner` 自动处理或手动实现 |
| **状态管理** | 区分 API 可见状态和内部状态 | messages 进入上下文，currentStep 保留内部 |
| **多步骤推理** | 将复杂任务分解为多个步骤 | 依赖前一步结果继续推理 |

#### 3. 状态管理的设计原则
```typescript
// ✅ 进入 API 上下文（与 AI 对话的内容）
messages: Message[]      // 对话历史
toolResults: ToolResult[] // 工具执行结果

// ❌ 内部使用（控制逻辑，不发送给 API）
currentStep: number      // 当前步骤
completed: boolean       // 完成状态
```

**设计原则**：
- 只有对话相关的信息进入 API 上下文
- 控制逻辑变量保留在 Agent 内部
- 节省 Token 成本，保护实现细节

#### 4. Agent 类型分类
- **信息检索型**：搜索、查询、整合信息
- **任务执行型**：API 调用、文件操作、数据处理
- **编程辅助型**：代码生成、执行、测试
- **领域专用型**：数据分析、客服、写作等

### 二、技术架构 (3 个)

#### 1. SDK 分层设计
```
Claude Code (完整 IDE)
    ↓
Agent SDK (高级抽象) - 工具编排、状态管理
    ↓
TypeScript SDK (基础 API) - 直接 API 访问
    ↓
Claude API (接口层)
```

#### 2. TypeScript SDK 核心组件
- **Messages API**：与 Claude 交互的主要方式
- **工具使用**：`betaZodTool`、`toolRunner`
- **流式响应**：Server Sent Events (SSE)
- **MCP 集成**：`mcpTools`、`mcpMessages` 辅助函数

#### 3. 运行时环境选择
| 运行时 | 用途 | TypeScript 支持 |
|--------|------|----------------|
| **Node.js** | 服务器端 | 需要编译或 ts-node |
| **Deno** | 现代运行时 | 原生支持 TS |
| **Bun** | 高性能运行时 | 原生支持 TS |
| **浏览器** | 客户端 | 需要打包 |

### 三、技术工具 (4 个)

#### 1. TypeScript 基础
- **定义**：JavaScript 的超集，添加静态类型检查
- **价值**：编译时错误检查、IDE 智能补全
- **核心概念**：Interface、类型注解、类型推导

#### 2. Zod Schema
- **定义**：运行时类型验证库
- **特点**：单一来源定义（既是类型又是验证器）
- **在 Agent 中的应用**：定义工具参数 schema，自动类型推导

```typescript
const weatherTool = betaZodTool({
  name: 'get_weather',
  inputSchema: z.object({
    location: z.string().describe('城市名称'),
    unit: z.enum(['celsius', 'fahrenheit']).default('celsius'),
  }),
  run: (input) => {
    // input 类型自动推断
    return getWeather(input.location, input.unit);
  },
});
```

#### 3. 模型参数配置
| 参数 | 说明 | 建议值 |
|------|------|--------|
| `model` | 模型选择 | `claude-sonnet-4-5-20250929` |
| `max_tokens` | 输出 token 限制 | 1024-4096（根据任务复杂度） |
| `temperature` | 温度（0-1） | 0-0.3（稳定）/ 0.7-1（创造性） |

#### 4. 本地模型集成方案
- **方案 1**：LangChain/Vercel AI SDK（统一接口）
- **方案 2**：OpenAI SDK + Ollama（兼容接口）
- **方案 3**：直接 HTTP 请求

### 四、实践技能 (5 个)

#### 1. SDK 安装和初始化
```bash
npm install @anthropic-ai/sdk
npm install zod
export ANTHROPIC_API_KEY=xxx
```

#### 2. 创建工具（Zod Schema 方式）
```typescript
import { betaZodTool } from '@anthropic-ai/sdk/helpers/beta/zod';

const tool = betaZodTool({
  name: 'tool_name',
  inputSchema: z.object({
    param: z.string(),
  }),
  description: '工具描述',
  run: async (input) => {
    // 工具逻辑
    return result;
  },
});
```

#### 3. 工具执行（自动循环）
```typescript
const result = await anthropic.beta.messages.toolRunner({
  model: 'claude-3-5-sonnet-20241022',
  max_tokens: 1024,
  messages: [{ role: 'user', content: '用户消息' }],
  tools: [tool1, tool2, tool3],
});
```

#### 4. API 错误处理
```typescript
try {
  await client.messages.create({...});
} catch (error) {
  if (error instanceof Anthropic.APIError) {
    console.log(error.status);  // 400, 401, 429, 500
    console.log(error.name);    // 错误类型
  }
}
```

#### 5. 工具错误处理
```typescript
import { ToolError } from '@anthropic-ai/sdk/lib/tools/BetaRunnableTool';

if (invalidInput) {
  throw new ToolError('详细的错误信息');
}
```

### 五、设计原则 (5 个)

1. **单一职责**：每个 Agent 专注一种任务
2. **工具组合**：简单工具组合实现复杂功能
3. **状态最小化**：只保存必要的状态
4. **可观察性**：提供清晰的日志
5. **优雅降级**：失败时提供备选方案
