## 什么是 OpenClaw

**OpenClaw 是一个真正能做事情的 AI。**

OpenClaw 是一个 AI 编排平台，运行在你的设备上、在你的渠道中、遵循你的规则。它是一个个人助手，易于使用，支持广泛的平台，并尊重隐私和安全。

### 项目演变历程

OpenClaw 经历了多个名称和形态的演变：
```
Warelay → Clawdbot → Moltbot → OpenClaw
```

这个演变过程反映了项目从个人学习项目到成熟 AI 编排平台的成长轨迹。

### 目标定位

打造一个个人助手，具备以下特点：

OpenClaw 是一个 **AI 编排系统**：提示词（prompts）、工具（tools）、协议（protocols）和集成（integrations）的协调平台。

## 技术选型

### 为什么选择 TypeScript？

OpenClaw 主要是一个编排系统，涉及：
- 提示词设计
- 工具定义
- 协议实现
- 系统集成

TypeScript 被选择的原因：
- ✅ **广泛认知**：开发者群体庞大
- ✅ **快速迭代**：开发效率高
- ✅ **易于阅读**：代码可读性好
- ✅ **易于修改**：扩展性强
- ✅ **默认可黑客化**：开放性强

## 核心特性

### 1. 多平台支持

OpenClaw 支持以下平台：
- 📱 **移动端**：iOS、Android
- 💻 **桌面端**：macOS、Windows、Linux
- 🌐 **Web**：Web UI 和 WebChat
- 💬 **消息渠道**：WhatsApp、Telegram、Discord、iMessage、Mattermost（插件）

### 2. 强大的插件系统

- **插件 API**：广泛而强大的插件 API
- **Core 保持精简**：核心保持轻量，可选功能作为插件提供
- **NPM 分发**：首选 NPM 包分发 + 本地扩展加载
- **ClawHub**：社区插件市场（clawhub.ai）

### 3. 内存系统

- **特殊插件槽**：内存是一个特殊的插件槽，同时只能激活一个内存插件
- **多种选项**：目前提供多种内存选项
- **未来方向**：计划收敛到一个推荐的默认路径

### 4. MCP 支持

OpenClaw 通过 `mcporter` 支持 MCP（Model Context Protocol）：
- **灵活性**：添加或更改 MCP 服务器无需重启 Gateway
- **精简核心**：保持核心工具/上下文表面精简
- **稳定性**：减少 MCP 变更对核心稳定性和安全性的影响

## 安全理念

### 安全设计原则

OpenClaw 的安全是**深思熟虑的权衡**：
- ✅ **强大的默认值**
- ✅ **不破坏能力**
- ✅ **有风险的路径需要明确说明**
- ✅ **由操作员控制**

### 权衡策略

**目标**：在保持强大实际工作能力的同时，使有风险的路径明确且由操作员控制。

## 当前不会合并的内容

为了保持项目聚焦和质量，以下内容目前不会被合并：

- ❌ 当可以在 ClawHub 上发布的新核心技能
- ❌ 完整文档翻译集（计划使用 AI 生成翻译）
- ❌ 不明显符合模型提供商类别的商业服务集成
- ❌ 围绕已支持渠道的包装渠道，没有明确的能力或安全差距
- ❌ 当 `mcporter` 已提供集成路径时，在核心中构建一流的 MCP 运行时
- ❌ 作为默认架构的 Agent 层次框架（管理者的管理者/嵌套规划器树）
- ❌ 重复现有 Agent 和工具基础设施的重型编排层

> **注意**：此列表是路线图护栏，不是物理定律。强烈的用户需求和强有力的技术理由可以改变它。

## 项目治理

### 贡献规则

- ✅ **一个 PR = 一个问题/主题**
- ✅ **不要捆绑多个不相关的修复/功能**
- ✅ **超过 5,000 行变更的 PR 只在特殊情况下审查**
- ✅ **不要一次打开大量微小的 PR**
- ✅ **鼓励将非常小的相关修复分组到一个专注的 PR 中**

### 文档规范

- 📚 **Mintlify**：文档托管在 Mintlify (docs.openclaw.ai)
- 🔗 **内部链接**：根目录相对路径，无 `.md`/`.mdx`
- 🌐 **国际化**：支持中文（zh-CN）文档生成
- 🔐 **安全文档**：`SECURITY.md` 包含规范的安全策略
