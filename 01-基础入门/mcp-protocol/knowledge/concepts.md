## 核心概念详解

### 1. 工具 (Tools)

**定义**：LLM 可以调用的可执行函数

**特征**：
- 有名称和描述
- 定义输入 JSON Schema
- 返回执行结果
- 可以声明权限和风险等级

**工具 Schema 示例**：
```json
{
  "name": "read_file",
  "description": "读取文件内容",
  "inputSchema": {
    "type": "object",
    "properties": {
      "path": {
        "type": "string",
        "description": "文件路径"
      }
    },
    "required": ["path"]
  }
}
```

**工具调用流程**：
```
LLM → 工具调用请求 → MCP Client → MCP Server → 执行工具
     ← 返回结果    ←           ←           ←
```

---

### 2. 资源 (Resources)

**定义**：LLM 可以读取的数据源

**类型**：

| 类型 | 说明 | 示例 |
|------|------|------|
| **静态资源** | 不变的内容 | README、配置文件 |
| **动态资源** | 实时生成的内容 | 日志、API 响应 |
| **可订阅资源** | 支持实时更新 | 数据库查询结果 |

**资源 URI 格式**：
```
resource://<server-name>/<resource-type>/<resource-id>
```

**示例**：
- `resource://filesystem/file:/path/to/file.txt`
- `resource://database/query:users`
- `resource://api/endpoint:status`

---

### 3. 提示 (Prompts)

**定义**：可复用的提示模板

**用途**：
- 标准化常见任务
- 确保提示一致性
- 减少重复编写

**提示结构**：
```json
{
  "name": "code_review",
  "description": "代码审查提示模板",
  "arguments": [
    {
      "name": "language",
      "description": "编程语言",
      "required": true
    },
    {
      "name": "focus",
      "description": "审查重点",
      "required": false
    }
  ]
}
```

## 传输层 (Transports)

### stdio 传输

**特点**：
- 最简单的传输方式
- 适合本地开发和测试
- 通过标准输入/输出通信

**使用场景**：
- 本地 MCP Server
- 命令行工具集成
- 开发调试

**配置示例**：
```json
{
  "command": "node",
  "args": ["server.js"],
  "env": {}
}
```

### HTTP 传输

**特点**：
- 支持远程连接
- 可通过代理
- 支持流式响应

**使用场景**：
- 远程 MCP Server
- 微服务架构
- 生产环境

**配置示例**：
```json
{
  "url": "https://api.example.com/mcp",
  "headers": {
    "Authorization": "Bearer xxx"
  }
}
```

## MCP 协议消息

### 请求-响应模型

所有 MCP 通信基于 JSON-RPC 2.0：

```json
// 请求
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "read_file",
    "arguments": {
      "path": "/path/to/file"
    }
  }
}

// 响应
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "content": "文件内容..."
  }
}
```

### 核心方法

| 方法 | 说明 |
|------|------|
| `initialize` | 初始化连接 |
| `tools/list` | 列出可用工具 |
| `tools/call` | 调用工具 |
| `resources/list` | 列出可用资源 |
| `resources/read` | 读取资源 |
| `prompts/list` | 列出可用提示 |
| `prompts/get` | 获取提示内容 |

## 权限与安全

### 权限模型

| 级别 | 说明 | 示例 |
|------|------|------|
| **读取** | 只读访问 | 读取文件、查询数据 |
| **写入** | 修改数据 | 写入文件、更新记录 |
| **执行** | 运行命令 | 执行脚本、调用 API |
| **危险** | 高风险操作 | 删除文件、系统更改 |

### 安全特性

- **显式授权**：危险操作需要用户确认
- **作用域限制**：限制可访问的资源范围
- **审计日志**：记录所有操作
- **传输安全**：支持 TLS/SSL

## MCP Server 生命周期

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  启动   │ -> │ 初始化  │ -> │  运行   │ -> │  关闭   │
│ Server  │    │ 握手    │    │ 服务    │    │ 清理    │
└─────────┘    └─────────┘    └─────────┘    └─────────┘
                   │                              │
                   ▼                              ▼
              交换能力                      保存状态
              建立连接                      释放资源
```

### 1. 启动阶段
- Server 启动
- 初始化资源
- 等待连接

### 2. 初始化阶段
- 接收 initialize 请求
- 返回 Server 能力（支持的协议版本、功能）
- 建立 Session

### 3. 运行阶段
- 处理工具调用
- 提供资源访问
- 管理提示模板

### 4. 关闭阶段
- 处理未完成的请求
- 保存状态（如需要）
- 释放资源

## 客户端-服务器交互模式

### 同步模式
```
Client → 请求 → Server
       ← 响应 ←
```

### 流式模式
```
Client → 请求 → Server
       ← 数据流 ←
       ← 数据流 ←
       ← 完成 ←
```

### 订阅模式
```
Client → 订阅 → Server
       ← 初始数据 ←
       ← 更新 ←
       ← 更新 ←
```
