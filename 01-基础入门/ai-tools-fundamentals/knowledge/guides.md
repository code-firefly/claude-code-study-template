# 使用指南与命令参考

> 来源：Claude Code 官方文档

## 核心命令

### 基础命令

| 命令 | 功能 | 示例 |
|------|------|------|
| `/help` | 显示帮助信息 | `/help` |
| `/edit <file>` | 编辑文件 | `/edit src/main.js` |
| `/read <file>` | 读取文件 | `/read README.md` |
| `/plan` | 进入计划模式 | `/plan` |

### 搜索命令

| 命令 | 功能 | 示例 |
|------|------|------|
| `/grep <pattern>` | 搜索文件内容 | `/grep "function test"` |
| `/glob <pattern>` | 搜索文件名 | `/glob "**/*.js"` |

### Git 命令

| 命令 | 功能 | 示例 |
|------|------|------|
| `/git status` | 查看 Git 状态 | `/git status` |
| `/git diff` | 查看文件差异 | `/git diff` |
| `/git commit` | 提交更改 | `/git commit` |

## 权限模式说明

### 默认模式
- ✅ 文件读取：自动允许
- ⚠️ 文件编辑：需要确认
- ⚠️ 命令执行：需要确认
- ❌ 危险操作：拒绝

### 自动模式
- ✅ 文件读取：自动允许
- ✅ 文件编辑：自动允许
- ⚠️ 命令执行：需要确认
- ❌ 危险操作：拒绝

## 最佳实践

### 1. 项目配置
- 始终在项目根目录创建 CLAUDE.md
- 包含清晰的项目说明
- 列出常用命令和注意事项

### 2. 权限管理
- 根据项目需求选择合适的权限模式
- 对危险操作保持警惕
- 定期审查操作日志

### 3. 工作流程
- 使用 `/plan` 模式规划复杂任务
- 分步骤执行，便于审查
- 利用书签系统追踪学习进度

### 4. 安全考虑
- 不要在配置文件中包含敏感信息
- 使用环境变量管理密钥
- 定期更新 Claude Code 到最新版本

## 常见问题解决

### Q: Claude Code 无法启动？
**A**: 检查：
1. 是否正确安装
2. 环境变量是否配置
3. 是否有足够的权限

### Q: 文件编辑被拒绝？
**A**:
1. 检查权限模式设置
2. 确认文件是否受保护
3. 查看错误信息了解原因

### Q: 如何查看所有可用命令？
**A**: 运行 `/help` 命令查看完整命令列表
