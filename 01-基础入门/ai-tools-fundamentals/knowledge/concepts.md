# 核心概念与架构

> 来源：Claude Code 文档 + 官方资源

## 核心概念

### 1. Tool Use（工具使用）

**Tool Use** 是 Claude Code 的核心能力之一。

**定义**：Claude 可以主动调用外部工具来完成特定任务，而非仅仅生成文本响应。

**工作原理**：
- Claude 分析用户请求
- 判断需要调用哪些工具
- 构造工具调用参数
- 执行工具调用
- 将结果整合到响应中

**示例工具**：
- 📝 文件编辑工具（Edit）
- 🔍 文件搜索工具（Glob、Grep）
- 💻 命令执行工具（Bash）
- 📚 代码读取工具（Read）
- 🌐 网络获取工具（WebFetch）

### 2. Hook（钩子）

**Hook** 是在特定事件发生时自动执行的脚本机制。

**触发时机**：
- 工具调用前（pre-hook）
- 工具调用后（post-hook）
- 用户提交编辑前

**用途**：
- 代码格式化
- 语法检查
- 自动化测试
- 自定义验证逻辑

### 3. 权限系统

Claude Code 使用细粒度的权限系统来控制 AI 可以执行的操作。

**权限类型**：
- **工具权限** - 控制哪些工具可以被调用
- **文件权限** - 控制 AI 可以访问的文件范围
- **命令权限** - 控制可以执行的命令类型

**权限模式**：
- **自动允许** - 某些安全的操作自动执行
- **用户确认** - 需要用户明确授权
- **拒绝** - 禁止执行某些危险操作

### 4. 配置系统

Claude Code 通过配置文件来自定义行为。

**配置层级**：
1. **全局配置** - `~/.claude/settings.json`
2. **项目配置** - `./CLAUDE.md`

**CLAUDE.md 内容**：
- 项目说明
- AI 工作流程指令
- 自定义命令定义
- 输出风格配置

## 架构组成

### 1. CLI 交互层
- 命令行界面
- 用户输入处理
- 输出格式化

### 2. Tool Use 引擎
- 工具调度
- 参数验证
- 结果处理

### 3. 权限管理系统
- 权限检查
- 用户提示
- 操作审计

### 4. 配置管理器
- 配置文件解析
- 环境变量处理
- 默认值管理

## 工作流程

```
用户输入
    ↓
CLI 解析
    ↓
Claude AI 处理
    ↓
判断需要工具
    ↓
权限检查 ← 用户确认（如需要）
    ↓
执行工具调用
    ↓
处理结果
    ↓
返回给用户
```

## 关键设计原则

### 1. 透明性
- 所有操作都向用户展示
- 明确显示工具调用过程
- 清晰的权限请求说明

### 2. 可控性
- 用户保留最终决策权
- 可以审查每个操作
- 支持拒绝或修改建议

### 3. 安全性
- 默认最小权限原则
- 危险操作需要明确授权
- 完整的操作审计日志

### 4. 可扩展性
- 支持自定义工具
- 支持自定义钩子
- 支持自定义命令（slash 命令）
