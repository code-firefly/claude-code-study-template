## 组件职责

### 1. 飞书 (Feishu)

**职责**：
- 用户交互界面
- 接收自然语言指令
- 展示执行结果（富文本格式）

**技术实现**：
- 飞书机器人 Webhook
- 消息接收与推送 API

---

### 2. OpenClaw (WSL 2)

**职责**：
- 飞书消息接收与解析
- 指令路由与调度
- 跨环境调用 Claude Code CLI
- 结果格式化与返回

**核心 Skills**：

#### 2.1 learning-manager Skill

```javascript
// 指令解析与路由
{
  "开始学习": "启动模块学习流程",
  "查看进度": "获取学习状态",
  "更新进度": "同步模块进度",
  "创建书签": "记录学习疑问",
  "继续书签": "继续探索书签",
  "完成书签": "完成当前书签"
}
```

#### 2.2 claude-code-skill Skill

```javascript
// Claude Code CLI 调用桥接
{
  "name": "claude-code",
  "execute": async (command) => {
    // 1. 路径转换 (WSL → Windows)
    // 2. 调用 claude-code.exe
    // 3. 解析返回结果
    // 4. 格式化输出
  }
}
```

---

### 3. Claude Code CLI (Windows)

**职责**：
- 执行学习命令

**工作流**：
- 读取项目文件

## 数据流设计

### 请求流

```
用户 → 飞书 → OpenClaw Webhook
  → 指令解析 → 路由到对应 Skill
  → 调用 Claude Code CLI
  → 文件操作 → 返回结果
```

### 响应流

```
Claude Code CLI → OpenClaw Skill
  → 结果格式化 → 飞书 Webhook 推送
  → 用户接收富文本消息
```

## 错误处理机制

### 1. 指令解析错误

```javascript
// learning-manager Skill
try {
  const intent = parseIntent(message);
  if (!intent) {
    return "未识别的指令，请参考帮助文档";
  }
} catch (error) {
  return "指令解析失败：" + error.message;
}
```

### 2. Claude Code 调用失败

```javascript
// claude-code-skill Skill
try {
  const result = await executeClaudeCode(command);
  return result;
} catch (error) {
  return {
    success: false,
    error: "Claude Code 执行失败",
    detail: error.message
  };
}
```

### 3. 文件操作错误

- 文件不存在：返回友好提示
- 权限不足：提示检查环境配置
- 格式错误：提示检查文件完整性

## 安全考虑

1. **指令白名单**：只允许预定义的学习命令
2. **路径验证**：防止路径遍历攻击
3. **错误隔离**：错误信息不暴露系统路径
4. **权限控制**：只允许操作学习项目目录

---

## 性能考虑

1. **结果缓存**：相同查询短时间内直接返回缓存
2. **异步处理**：长时间操作异步执行
3. **超时控制**：Claude Code 调用设置超时
